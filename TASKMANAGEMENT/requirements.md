## 1. プロジェクト概要

### 1.1 背景・目的

チームや個人のタスクを視覚的に管理するため、Trello 風のカンバンボードアプリを開発する。
カード（タスク）をカラム間でドラッグ&ドロップして進捗状況を直感的に把握できることを目的とする。

### 1.2 対象ユーザー

- タスクを視覚的に整理・管理したい個人ユーザー

### 1.3 スコープ

**対象範囲:**
- カラム（ステータス列）の管理
- カード（タスク）の管理
- カード間のドラッグ&ドロップ移動
- データのバックエンドDB永続化

**対象外:**
- ユーザー認証・ログイン機能
- 複数ユーザーによる共同編集
- 通知機能
- ファイル添付
- コメント機能

## 4. ユースケース

### 4.1 アクター

| アクター | 説明 |
|----------|------|
| ユーザー | アプリを利用してタスクを管理する人 |

### 4.2 ユースケース一覧

| ID | ユースケース名 | アクター |
|----|---------------|----------|
| UC-01 | カラムを追加する | ユーザー |
| UC-02 | カラムを削除する | ユーザー |
| UC-03 | カードを追加する | ユーザー |
| UC-04 | カードを編集する | ユーザー |
| UC-05 | カードを削除する | ユーザー |
| UC-06 | カードを移動する（ドラッグ&ドロップ） | ユーザー |

### 4.3 ユースケース詳細

#### UC-01: カラムを追加する

- **事前条件:** ボード画面が表示されている
- **基本フロー:**
  1. ユーザーが「カラム追加」ボタンをクリックする
  2. カラム名入力フィールドが表示される
  3. ユーザーがカラム名を入力して確定する
  4. 新しいカラムがボードの右端に追加される
- **代替フロー:** カラム名が空欄の場合、追加されずに入力フィールドが閉じる
- **事後条件:** 新しいカラムがDBに保存され、ボードに表示される

---

#### UC-02: カラムを削除する

- **事前条件:** 削除対象のカラムが存在する
- **基本フロー:**
  1. ユーザーがカラムの削除ボタンをクリックする
  2. 確認ダイアログが表示される
  3. ユーザーが削除を確定する
  4. カラムおよび配下のカードが削除される
- **代替フロー:** ユーザーがキャンセルした場合、削除されない
- **事後条件:** カラムと配下のカードがDBから削除され、ボードから消える

---

#### UC-03: カードを追加する

- **事前条件:** 追加先のカラムが存在する
- **基本フロー:**
  1. ユーザーがカラム内の「カード追加」ボタンをクリックする
  2. タイトル入力フィールドが表示される
  3. ユーザーがタイトルを入力して確定する
  4. カードがカラムの末尾に追加される
- **代替フロー:** タイトルが空欄の場合、追加されずに入力フィールドが閉じる
- **事後条件:** 新しいカードがDBに保存され、カラムに表示される

---

#### UC-04: カードを編集する

- **事前条件:** 編集対象のカードが存在する
- **基本フロー:**
  1. ユーザーがカードをクリックする
  2. タイトルが編集可能な状態になる
  3. ユーザーがタイトルを変更して確定する
  4. カードのタイトルが更新される
- **代替フロー:** タイトルが空欄の場合、変更が破棄される
- **事後条件:** 変更内容がDBに保存される

---

#### UC-05: カードを削除する

- **事前条件:** 削除対象のカードが存在する
- **基本フロー:**
  1. ユーザーがカードの削除ボタンをクリックする
  2. カードが即時削除される
- **事後条件:** カードがDBから削除され、カラムから消える

---

#### UC-06: カードを移動する

- **事前条件:** カードが1枚以上存在し、カラムが2列以上ある
- **基本フロー:**
  1. ユーザーがカードをドラッグする
  2. 移動先のカラムにドロップする
  3. カードが移動先カラムに移動する
- **代替フロー:** 同一カラム内でドロップした場合、カラム内での並び順が変わる
- **事後条件:** 移動後の位置（column_id・position）がDBに保存される

---

## 5. 機能要件

### 5.1 カラム管理
- 初期カラムとして「Todo」「In Progress」「Done」の3列を表示する
- カラムの追加ができる（タイトルを入力して作成）
- カラムの削除ができる（配下のカードも同時に削除）

### 5.2 カード管理
- カードの追加（タイトルを入力して作成）
- カードの削除
- カードの編集（タイトルの変更）

### 5.3 ドラッグ&ドロップ
- カードをカラム間でドラッグ&ドロップして移動できる
- カラム内での並び替えができる

### 5.4 データ保存
- カラム・カードの状態はバックエンドDBに保存する
- ページをリロードしてもデータが保持される

## 6. 非機能要件

| 項目 | 要件 |
|------|------|
| アーキテクチャ | SPA（シングルページアプリケーション） |
| レスポンシブ | デスクトップのみ（モバイル対応は対象外） |
| ブラウザ対応 | Google Chrome 最新版 |
| パフォーマンス | 通常操作（カード追加・移動等）は1秒以内に反映 |
| セキュリティ | HTTPS通信必須。フロントエンドでの入力値バリデーションを実施する |
| エラーハンドリング | APIリクエスト失敗時はユーザーにエラーメッセージを表示する |
| 保守性 | ソースコードはGitで管理する |
| 可用性 | デプロイ先確定後に定義する |

## 8. ER図

### 8.1 テーブル一覧

| テーブル名 | 説明 |
|-----------|------|
| columns | カラム（ステータス列）を管理する |
| cards | カード（タスク）を管理する |

### 8.2 ER図

```
┌─────────────────────────┐       ┌──────────────────────────┐
│         columns         │       │          cards           │
├─────────────────────────┤       ├──────────────────────────┤
│ PK  id          INTEGER │──┐    │ PK  id          INTEGER  │
│     title       VARCHAR │  └──< │ FK  column_id   INTEGER  │
│     position    INTEGER │       │     title       VARCHAR  │
│     created_at  DATETIME│       │     position    INTEGER  │
│     updated_at  DATETIME│       │     created_at  DATETIME │
└─────────────────────────┘       │     updated_at  DATETIME │
                                  └──────────────────────────┘
```

### 8.3 テーブル定義

#### columns テーブル

| カラム名 | 型 | NOT NULL | 説明 |
|---------|-----|----------|------|
| id | INTEGER | ○ | 主キー（自動採番） |
| title | VARCHAR(100) | ○ | カラム名 |
| position | INTEGER | ○ | 表示順（左から何番目か） |
| created_at | DATETIME | ○ | 作成日時 |
| updated_at | DATETIME | ○ | 更新日時 |

#### cards テーブル

| カラム名 | 型 | NOT NULL | 説明 |
|---------|-----|----------|------|
| id | INTEGER | ○ | 主キー（自動採番） |
| column_id | INTEGER | ○ | 外部キー（columns.id） |
| title | VARCHAR(255) | ○ | カード（タスク）名 |
| position | INTEGER | ○ | カラム内での表示順 |
| created_at | DATETIME | ○ | 作成日時 |
| updated_at | DATETIME | ○ | 更新日時 |

**制約:**
- `cards.column_id` は `columns.id` を参照（外部キー制約）
- カラム削除時、配下のカードはカスケード削除する
