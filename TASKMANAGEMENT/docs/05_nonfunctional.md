# 6. 非機能要件

| 項目 | 要件 |
|------|------|
| アーキテクチャ | SPA（シングルページアプリケーション） |
| レスポンシブ | デスクトップのみ（モバイル対応は対象外） |
| ブラウザ対応 | Google Chrome 最新版 |
| パフォーマンス | 通常操作（カード追加・移動等）は1秒以内に反映 |
| セキュリティ | HTTPS通信必須。フロントエンドでの入力値バリデーションを実施する |
| エラーハンドリング | APIリクエスト失敗時はユーザーにエラーメッセージを表示する |
| 保守性 | ソースコードはGitで管理する |
| 可用性 | デプロイ先確定後に定義する |
