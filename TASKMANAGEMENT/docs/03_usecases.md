# 4. ユースケース

### 4.1 アクター

| アクター | 説明 |
|----------|------|
| ユーザー | アプリを利用してタスクを管理する人 |

### 4.2 ユースケース一覧

| ID | ユースケース名 | アクター |
|----|---------------|----------|
| UC-01 | カラムを追加する | ユーザー |
| UC-02 | カラムを削除する | ユーザー |
| UC-03 | カードを追加する | ユーザー |
| UC-04 | カードを編集する | ユーザー |
| UC-05 | カードを削除する | ユーザー |
| UC-06 | カードを移動する（ドラッグ&ドロップ） | ユーザー |

### 4.3 ユースケース詳細

#### UC-01: カラムを追加する

- **事前条件:** ボード画面が表示されている
- **基本フロー:**
  1. ユーザーが「カラム追加」ボタンをクリックする
  2. カラム名入力フィールドが表示される
  3. ユーザーがカラム名を入力して確定する
  4. 新しいカラムがボードの右端に追加される
- **代替フロー:** カラム名が空欄の場合、追加されずに入力フィールドが閉じる
- **事後条件:** 新しいカラムがDBに保存され、ボードに表示される

---

#### UC-02: カラムを削除する

- **事前条件:** 削除対象のカラムが存在する
- **基本フロー:**
  1. ユーザーがカラムの削除ボタンをクリックする
  2. 確認ダイアログが表示される
  3. ユーザーが削除を確定する
  4. カラムおよび配下のカードが削除される
- **代替フロー:** ユーザーがキャンセルした場合、削除されない
- **事後条件:** カラムと配下のカードがDBから削除され、ボードから消える

---

#### UC-03: カードを追加する

- **事前条件:** 追加先のカラムが存在する
- **基本フロー:**
  1. ユーザーがカラム内の「カード追加」ボタンをクリックする
  2. タイトル入力フィールドが表示される
  3. ユーザーがタイトルを入力して確定する
  4. カードがカラムの末尾に追加される
- **代替フロー:** タイトルが空欄の場合、追加されずに入力フィールドが閉じる
- **事後条件:** 新しいカードがDBに保存され、カラムに表示される

---

#### UC-04: カードを編集する

- **事前条件:** 編集対象のカードが存在する
- **基本フロー:**
  1. ユーザーがカードをクリックする
  2. タイトルが編集可能な状態になる
  3. ユーザーがタイトルを変更して確定する
  4. カードのタイトルが更新される
- **代替フロー:** タイトルが空欄の場合、変更が破棄される
- **事後条件:** 変更内容がDBに保存される

---

#### UC-05: カードを削除する

- **事前条件:** 削除対象のカードが存在する
- **基本フロー:**
  1. ユーザーがカードの削除ボタンをクリックする
  2. カードが即時削除される
- **事後条件:** カードがDBから削除され、カラムから消える

---

#### UC-06: カードを移動する

- **事前条件:** カードが1枚以上存在し、カラムが2列以上ある
- **基本フロー:**
  1. ユーザーがカードをドラッグする
  2. 移動先のカラムにドロップする
  3. カードが移動先カラムに移動する
- **代替フロー:** 同一カラム内でドロップした場合、カラム内での並び順が変わる
- **事後条件:** 移動後の位置（column_id・position）がDBに保存される
